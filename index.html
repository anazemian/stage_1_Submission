Lesson 1: HTML and Web Basics

What is HTML and how does the Web work?

HTML stands for Hyper Text Markup Language. The web works in conjuction with HTML, sending a request via http through the internet to a server that processes that request.

The server then finds the appropriate HTML document and sends it back to the users computer where a browser such as Google Chrome will display that document in a user friendly manner for the person using the internet.

Here is a hilariously outdated video that explains how the web works for kids.

What makes HTML?

HTML is made of 3 basic parts. These parts are as follows:

  • Text content: what you see
  • Mark-up: what it looks like
  • References to other documents: i.e. images and videos
What can HTML do?

HTML is made up of what is called HTML Elements. When writing HTML, we tell browsers the type of each element by using HTML tags. These tags are capable of doing such things as adding images and links on our webpage.

Tags also help design the webpage with attributes like writing in bold or italics.

Inline vs. Block elements

We also learn the difference between inline and block elements.
Inline does not create a new block of content, such as the span tag, or the break tag, where as Block will create a new block of content, such as the paragraph tag, or the div tag. Block elements form an "invisible box" around the content inside of them.

How smart is a computer...really?

Finally, we also learn computers are "stupid". Not in the sense that they cannot complete simple tasks, but in that the smallest mistake can cause the biggest of issues. However, if we learn how to speak/code correctly, we can make computers do amazing things.

Lesson 2: Structured documents and HTML

Developer Tools(in the browser)

Using a browser such as chrome will allow you to open the developer tools and see the code for any website. Within these developer tools, you can see the Document Object Model or DOM. The DOM is the tree like structure that is the breakdown of the HTML elements. If it were to be drawn out on paper, it would be like seeing the family tree of a website.

HTML, CSS, and JS working together

HTML, CSS, and Java/js, all work together to create a more structured webpage. If thought of together in the context of a house, HTML would be the frame or the walls, CSS would be the paint and decorations, and js would be the TV remote, or garage door opener. These three work together to make one cohesive home, or in this case, website that allows the site to be user friendly. The more these elements work together, the more fluid and smooth the experience will be for the end user.

Trees, Boxes, and Indentation

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, etc...

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is.

Text editors

Text editors such as Sublime text, or CodePen, allow people to code with much more ease. These are editors specifially designed to code, where as something like notepad or MS Word are just for word processing.

Lesson 3:Adding style to HTML structure with CSS

Avoiding repetition

It is very important for numerous reasons we avoid repetition.

Avoiding Errors: When programmers don't avoid repetition, they often have to do the same things over and over. This may result in copy and pasting certain style attributes into many HTML elements, or rewriting the same code, perhaps sometimes with a very slight difference, many times over. This can lead to errors when the programer decides to make a change. If they don't make that same change everywhere the repeated code appears, it will lead to problems.

Consistency: When we avoid repetition, and work smarter, we can make changes at a much quicker rate, rather than maunally making a change to every section of our code.

What is CSS?

CSS stands for Cascading Style Sheets and they give programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.

3 ways to add CSS

There are three ways to add CSS to your work.

Writing CSS into your head element. This is good for smaller projects.

Link HTML to a separate CSS file. This adds an extra step to the process but is more organized for a larger project.

Inline with code. This is generally a bad idea as it leads to a lot of repetion in coding.
